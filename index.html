Chapter 2 - Model and Cost Function

Model Representation

Training set: Number of training examples (m)/ input variable (x)/ output variable (y)

How do we represent h?

Linear regression with one variable (Univariate linear regression): h(x) = a+bx

Cost Function

Choose a,b so that h(x) is close to y for your training examples (x, y).

Cost function:

Hypothesis: H(x) = θ0 +θ1x

Parameters: θ0, θ1

Cost function

Goal: Minimize J(θ0，θ1)

Squared error function is the most commonly used one for regression problems.

Hypothesis function is a function of x, cost function J(θ) is a function of the parameter θ0 and θ1

Contour figure:

Gradient Descent Algorithm:

repeat until convergence

α is the learning rate, deciding the step we use. We need to simultaneously update θ0 and θ1 for a correct implementation. Use all parameters before update them

α太小，会导致收敛速度较慢。α太大则可能不能收敛。

Gradient descent can converge to a local minimum, even with fixed α.

“Batch” Gradient Descent: Each step of gradient descent uses all the training examples.

Chapter 3 - Multivariate Linear Regression

Multiple Features

Hypothesis:

Previously:
$$
h_{\theta}(x)=\theta_0+\theta_{1}x
$$
Now:
$$
h_{\theta}(x)=\theta_0+\theta_{1}x_{1}+\theta_{2}x_{2}+…+\theta_{n}x_{n}
$$
For convenience, define x0=1, the get follow vetors:
$$
x=\left[
\begin{matrix}
x_1 \
x_2 \
x_3 \
\vdots \
x_n
\end{matrix}
\right]
<=R^{n+1}
$$

$$
\theta=\left[
\begin{matrix}
\theta_1 \
\theta_2 \
\theta_3 \
\vdots \
\theta_n
\end{matrix}
\right]
<=R^{n+1}
$$

And:
$$
h_\theta(x)=
\left[
\begin{matrix}
\theta_0\theta_1…\theta_n
\end{matrix}
\right]
\left[
\begin{matrix}
x_1 \
x_2 \
x_3 \
\vdots \
x_n
\end{matrix}
\right]=\theta^Tx
$$

Gradient Descent for Multiple Variables

Practice I - Feature Scaling(特征缩放)

Idea: Make sure features are on a similar scale.

E.g. x1 = size(0-2000 feet^2), x2 = number of bedrooms(1-5).

It is more efficient to find the global minimum

Implementation:

Get every feature into approximately a -1~1 range. (归一化？)

Mean normalization:

​ Replace xi with xi-μi to make features have approximately zero mean.

​ xi = (xi-μi)/si

​ μi is the average value of xi in the training set. si is the range of xi

Practice II - Learning Rate

Goal:

  1. Debugging: how to make sure gradient descent is working correctly.
  2. How to choose learning rate α

For Goal 1:

J(θ) should decrease after every iteration, use automatic converge test.

Declare convergence if J decreases by less than 10^(-3) in one iteration.

If gradient descent not working, use smaller α.

For sufficiently small α, J should decrease on every iteration. But if α is too small, the convergence might be very slow.

Choice of features:

We can improve our features and the form of our hypothesis function in a couple of different ways.

We can combine multiple features into one. For example, wen can combine x1 and x2 into a new feature x3 by taking x1*x2.

For Polynomial Regression, we can change the behavior of curve of your hypothesis function by making it quadratic, cubic or square root function and so on. If do so, the feature scaling is very important.

Normal Equation

$$
m\ examples:(x^1,y^1),…,(x^m,y^m);\ n features.
\x^i=\left[
\begin{matrix}
x_0^i\
x_1^i\
x_3^i\
…\
x_n^i
\end{matrix}
\right]
\in R
$$

$$
X(desgin\ matrix)=
\left[
\begin{matrix}
-(X^{(1)})^T\
-(X^{(2)})^T\
-(X^{(3)})^T\
…\
-(X^{(m)})^T\
\end{matrix}
\right]
$$

$$
\theta = (X^TX)^{-1}X^Ty
$$

Comparison:

  • Gradient Descent
    • Need to choose α
    • Needs many iterations
    • Works well even when n is large
  • Normal Equation
    • No need to choose α
    • Don’t need to iterate
    • Need to compute $(X^TX)^{-1}$
    • Slow if n is very large. The complexity of computing the inversion is $O(n^3)$. So if the features are large, the normal equation will be slow. In practice, if the n exceeds 10000, it might be good to go to an iterative process.

Normal Equation Noninvertibility (optional)

If $X^TX$ is non-invertible (singular/degenerate), (this is rare), such as

  • Redundant features (linearly dependent)
  • Too many features (e.g. m<=n)
    • delete some features, or use regularization
